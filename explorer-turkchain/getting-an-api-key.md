# Getting an API Key

TurkChain Explorer provides an API that allows developers to programmatically access onchain data such as blocks, transactions, addresses, tokens, and smart contracts.

To use the TurkChain Explorer API, you need to create an account and generate an API key.

***

### 1. Create a TurkChain Explorer Account

First, create a free account on TurkChain Explorer.

* Visit: [https://explorer.turkchain1919.com](https://explorer.turkchain1919.com)
* Click **Sign Up** or **Register**
* Complete the account registration process

An account is required to generate and manage API keys.

***

### 2. Generate an API Key

After logging in:

1. Open your **Account Dashboard**
2. Navigate to the **API Dashboard**
3. Click **“Add +”** to create a new API key

Once created, your API key can be used to access all supported API endpoints provided by TurkChain Explorer.

> **Note:**\
> API keys are personal. Do not share your key publicly or embed it directly in client-side applications.

***

### 3. Make Your First API Request

With your API key, you can start making requests to the TurkChain Explorer API.

You can:

* Test endpoints using the **API Playground** available in these docs
* Integrate the API into your backend services
* Query blockchain data programmatically

Example use cases include:

* Fetching transaction details
* Monitoring address activity
* Verifying smart contracts
* Indexing onchain data for analytics or dashboards

***

### Remix Contract Verification

If you use **Remix** to develop smart contracts, you can verify deployed contracts directly using the TurkChain Explorer API.

* Deploy your contract to TurkChain
* Use the **Verify & Publish** feature on the explorer
* Or verify via supported API endpoints

Verified contracts improve transparency and trust for users interacting with your project.

***

### Supported Chains

The TurkChain Explorer API supports:

* **TurkChain 1919** (Chain ID: 1919)

TurkChain is a single-network blockchain.\
There are no separate mainnet or testnet environments unless explicitly announced.
